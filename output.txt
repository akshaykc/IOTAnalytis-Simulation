1) The key observation in this experiment is that Mean D value significanlty decreases as the size of the wait buffer is increased. When we are not changing the interarrival time and the service time 
and vary the buffer size repeatedly we can see that lesser and lesser IoT devices will be rejected and more devices will be added to the buffer since there is more space.

2) This can be observed in the graph "Task 2.png" where the Mean D value falls quickly. As the buffer size increases lesser and lesser number of IoT devices will be rejected and the orbitting time(D)
will reduce

3) From the graph 'Task 2 mean T.png' we can observe that the T value decreseas rapidly when the buffer size is changed from 1 to 3. This is because when the buffer size is one there are no devices
in Queue to immediately be serviced and the server might end up being idle. When the buffer size is 1 the devices orbit around and arrive at a later time even if the server is actually free. This 
drastically increases the average time taken by an IoT device to be recertified. If the buffer size is larger there will be devices waiting for service and immediately occupy the server without 
wasting server time.

Further increase of buffer size from 3 does not show any improvement in T value since the buffer has reached an optimal value.
